# Navi-UI - Project Specification

**Nombre del proyecto:** Navi-UI  
**Paquete npm:** `@navi-ui/react`  
**Descripción:** Librería moderna de componentes React + TypeScript con Storybook, enfocada en accesibilidad WCAG 2.2 AA, theming Light/Dark y rendimiento.

## Tech Stack (2026)
- React 19 (con soporte RSC)
- TypeScript 5.8+ (strict mode)
- Vite 6+
- Storybook 8.5+ / 10.x
- Tailwind CSS v4
- Radix UI (primitivas headless)
- clsx + tailwind-merge
- tsup para build de librería
- Vitest + React Testing Library
- Chromatic para visual testing

Razones principales:

kebab-case es la convención estándar en npm para paquetes con varias palabras (casi todas las librerías modernas lo usan: react-router, tanstack-query, next-themes, shadcn-ui, etc.).
npm desaconseja nombres en camelCase para paquetes públicos porque puede causar problemas de case-sensitivity en diferentes sistemas operativos.
Es mucho más legible y profesional: la gente entiende inmediatamente que es "Navi UI" (Navigation + UI o tu marca personal).

**Nombre del proyecto:** Navi-UI  
**Paquete npm:** `@navi-ui/react`  
**Descripción:** Librería moderna de componentes React + TypeScript con Storybook, enfocada en accesibilidad WCAG 2.2 AA, theming Light/Dark y rendimiento.

- React 19 (con soporte RSC)
- TypeScript 5.8+ (strict mode)
- Vite 6+
- Storybook 8.5+ / 10.x
- Tailwind CSS v4
- Radix UI (primitivas headless)
- clsx + tailwind-merge
- tsup para build de librería
- Vitest + React Testing Library
- Chromatic para visual testing

## Reglas de Desarrollo (importante que Cursor siga)

1. Todos los componentes deben ser:
   - Funciones con `forwardRef`
   - 100% TypeScript con tipos estrictos
   - Usar el helper `cn()` para combinar clases
   - Soporte completo de dark mode
   - Accesibilidad WCAG 2.2 AA (ARIA labels, keyboard navigation)
   - Estados: default, hover, focus, disabled, loading, error

2. Usa siempre Radix UI como base cuando sea posible (para accesibilidad).

3. Nombre de archivos:
   - Componentes: `Button.tsx`, `Button.stories.tsx`
   - Utils: `cn.ts`

4. El helper `cn()` debe estar en `src/utils/cn.ts`

5. Tailwind v4 se importa con `@import "tailwindcss";` en `src/index.css`
